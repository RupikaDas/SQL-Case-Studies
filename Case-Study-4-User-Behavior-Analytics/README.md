# Case Study 4: User Behavior Analytics

## Overview

This case study simulates a product analytics workflow focused on user engagement, activity tracking, subscription behavior, and retention analysis.

Using SQL, the project investigates user sessions, platform activities, subscription conversion, engagement scoring, and behavioral trends to support product and growth analytics decision-making.

---

## Business Problem

Digital platforms rely on understanding user behavior to improve engagement, increase retention, and drive subscription growth.

The objective of this project is to demonstrate how SQL can be used to:

* Analyze user engagement
* Measure activity levels
* Identify high-value users
* Monitor subscription performance
* Detect churn-risk users
* Support product analytics initiatives

---

## Dataset

The database consists of four tables:

### Users

Stores user profile information.

### Sessions

Stores user session activity and duration.

### Activities

Stores user interactions such as:

* Login
* Like
* Comment
* Share
* Purchase

### Subscriptions

Stores subscription plan and status information.
